# Research Data Analysis Skill

## 适用场景

当用户提出以下任务时，优先应用本 Skill：

- 分析 CSV、TSV、Excel 导出的文本数据或实验结果文件；
- 对科研数据做描述性统计、分组比较、相关性分析或趋势分析；
- 清洗缺失值、异常值、重复样本、列名不规范等问题；
- 生成可复现实验分析脚本；
- 将分析结果整理成论文、实验报告或答辩可用的解释。

## 工作流程

1. 明确数据目标
   - 先确认用户要回答的科研问题，例如“比较两组差异”“分析指标相关性”“找异常样本”。
   - 如果目标不清楚，先给出最小必要问题，不要直接编造分析结论。

2. 检查数据结构
   - 优先使用 `ls` / `find` 定位数据文件；
   - 使用 `read` 查看文件头部、列名和少量样本；
   - 如果文件较大，先读取小片段，不要一次性把全部内容放入上下文。

3. 制定分析计划
   - 说明将做哪些步骤：数据加载、字段检查、缺失值统计、基础统计、核心分析、结果解释；
   - 对每一步给出原因，避免只给代码不解释。

4. 编写可复现代码
   - 优先生成 Python 脚本；
   - 常用库优先级：`pandas`、`numpy`、`scipy`、`matplotlib`；
   - 如果环境缺少库，先给出替代方案或安装建议，不要假设一定可用。

5. 验证与解释
   - 运行脚本或给出明确运行命令；
   - 输出关键统计量、样本数、缺失比例、主要结论；
   - 区分“数据支持的结论”和“需要进一步实验验证的推断”。

## 输出要求

- 先给分析结论或执行计划，再给代码；
- 结果解释要使用中文；
- 面向科研场景时，说明统计方法的适用条件和局限；
- 如果用户准备面试或答辩，额外给一段可直接表达的话术。

## 边界与注意事项

- 不要伪造数据、统计结果或显著性结论；
- 没有读取到数据前，不要声称已经完成分析；
- 不要把相关性说成因果关系；
- 对样本量很小、缺失严重或分布异常的数据，要明确提醒风险；
- 对涉及隐私或敏感数据的文件，只做必要摘要，不输出原始敏感内容。

## 示例触发语句

- “帮我分析这个 csv 数据”
- “对实验结果做统计分析”
- “看看两组数据有没有差异”
- “帮我写一个科研数据清洗脚本”
- “把分析结果整理成论文里的结果描述”
